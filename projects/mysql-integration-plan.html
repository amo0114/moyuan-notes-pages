MySQL Integration Implementation Plan ​

For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

Goal: Replace ConcurrentHashMap in-memory storage in stock-service (库存 + 流水) and order-service (订单) with MySQL persistence via MyBatis-Plus, introducing OPTIMISTIC + PESSIMISTIC dual lock strategies switchable via Nacos hot-config, with REQUIRES_NEW audit logging for conflicts and Testcontainers integration tests.

Architecture: 一实例多 schema (stock_db + order_db) on local MySQL 8.x; MyBatis-Plus 3.5.9 ORM; Flyway-managed migrations per service; HikariCP connection pool; strategy pattern in stock-service for lock-mode switching; REQUIRES_NEW audit via separate Bean to avoid Spring AOP self-invocation; existing StockApi HTTP contract & StockService.deductStock Java contract preserved unchanged (returns Integer: null=not-found, -1=insufficient, >=0=remaining).

Tech Stack: Java 17, Spring Boot 3.5.14, Spring Cloud Alibaba 2025.0.0.0, MyBatis-Plus 3.5.9, MySQL 8.x, Flyway, HikariCP, JUnit 5, Mockito, Testcontainers 1.20.x.

Spec: docs/superpowers/specs/2026-05-27-mysql-integration-design.md

package com.snowvictor.cloudwork.stock.service.deduct;

/**
 * 扣减结果不可变值对象。
 *
 * resultCode 语义：
 *   200 = 成功
 *   -1  = 库存不足
 *   404 = 商品不存在
 *   409 = 乐观锁冲突（仅在外层 try/catch 路径生成，不通过本对象返回）
 *
 * remainingStock 在 success 时为新库存，其他情况下为 null。
 */
public record DeductOutcome(int resultCode, Integer remainingStock, String message) {

    public static DeductOutcome success(int remainingStock) {
        return new DeductOutcome(200, remainingStock, "OK");
    }

    public static DeductOutcome insufficient() {
        return new DeductOutcome(-1, null, "INSUFFICIENT_STOCK");
    }

    public static DeductOutcome notFound() {
        return new DeductOutcome(404, null, "PRODUCT_NOT_FOUND");
    }
}

            throw new OptimisticLockConflictException(
                    "optimistic conflict on productId=" + productId);
        }
        return DeductOutcome.success(item.getStock());
    }

package com.snowvictor.cloudwork.stock.service.deduct;

import com.snowvictor.cloudwork.stock.exception.UnsupportedLockModeException;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.cloud.context.config.annotation.RefreshScope;
import org.springframework.stereotype.Component;

import java.util.List;
import java.util.Locale;
import java.util.Map;
import java.util.stream.Collectors;

@Component
@RefreshScope
public class StockDeductStrategyResolver {

    @Value("${stock.deduct.lock-mode:OPTIMISTIC}")
    private String mode;

    private final Map<String, StockDeductStrategy> byMode;

    public StockDeductStrategyResolver(List<StockDeductStrategy> strategies) {
        this.byMode = strategies.stream()
                .collect(Collectors.toUnmodifiableMap(s -> s.mode().name(), s -> s));
    }

    /** Setter exposed for tests; in runtime Spring fills `mode` via @Value. */
    void setMode(String mode) {
        this.mode = mode;
    }

    public StockDeductStrategy resolve() {
        String key = (mode == null) ? "" : mode.trim().toUpperCase(Locale.ROOT);
        StockDeductStrategy strategy = byMode.get(key);
        if (strategy == null) {
            throw new UnsupportedLockModeException("unsupported lock mode: " + mode);
        }
        return strategy;
    }
}

import com.snowvictor.cloudwork.stock.service.deduct.StockDeductStrategy;
import com.snowvictor.cloudwork.stock.service.deduct.StockDeductStrategyResolver;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
@Slf4j
public class StockService {

    private final StockDeductStrategyResolver resolver;
    private final StockDeductLogMapper logMapper;
    private final StockDeductAuditService auditService;   // 独立 Bean, 避免 self-invocation

    /**
     * 扣减库存。HTTP 契约（与原 StockController 配合）保持不变：
     *   return null  → 商品不存在 (StockController 转 HTTP 404)
     *   return -1    → 库存不足   (StockController 转 HTTP 400)
     *   return >= 0  → 扣减成功, 值为剩余库存
     * 冲突 / 非法参数 / 配置错误均通过异常抛出，由 StockServiceExceptionHandler 处理。
     */
    @Transactional(rollbackFor = Exception.class)
    public Integer deductStock(String productId, Integer qty, String traceId) {
        // 1) 入口校验, 不进入策略层, 不写流水
        if (qty == null || qty <= 0) {
            throw new InvalidDeductRequestException("quantity must be positive: " + qty);
        }
        // 2) 解析策略 (Nacos 配错时抛 UnsupportedLockModeException)
        StockDeductStrategy strategy = resolver.resolve();
        long start = System.currentTimeMillis();
        try {
            DeductOutcome outcome = strategy.deduct(productId, qty);
            // 3) 主事务同事务写流水 (success / insufficient / notFound 都走这里)
            logMapper.insert(toLog(traceId, productId, qty, outcome,
                                   strategy.mode().name(),
                                   System.currentTimeMillis() - start));
            return toLegacyReturn(outcome);
        } catch (OptimisticLockConflictException e) {
            // 4) 仅冲突场景: 主事务回滚, 独立 Bean 用 REQUIRES_NEW 写失败流水
            auditService.writeConflictLog(traceId, productId, qty, strategy.mode());
            throw e;
        }
    }

    private StockDeductLog toLog(String traceId, String productId, int qty,
                                 DeductOutcome out, String mode, long costMs) {
        StockDeductLog log = new StockDeductLog();
        log.setTraceId(traceId);
        log.setProductId(productId);
        log.setQuantity(qty);
        log.setResultCode(out.resultCode());
        log.setRemainingStock(out.remainingStock());
        log.setLockMode(mode);
        log.setCostMs((int) costMs);
        return log;
    }

    /** 翻译 DeductOutcome 回旧 StockController 期望的 Integer 三态。 */
    private Integer toLegacyReturn(DeductOutcome outcome) {
        return switch (outcome.resultCode()) {
            case 200 -> outcome.remainingStock();
            case -1  -> -1;
            case 404 -> null;
            default  -> throw new IllegalStateException(
                    "unexpected outcome: " + outcome.resultCode());
        };
    }
}

        log.error("unsupported lock mode: {}", e.getMessage());

        Map<String, Object> body = new LinkedHashMap<>();
        body.put("code", status.value());
        body.put("message", message);
        body.put("data", null);
        return ResponseEntity.status(status).body(body);
    }

package com.snowvictor.cloudwork.order.service;

import com.snowvictor.cloudwork.api.stock.dto.StockDeductResponse;
import com.snowvictor.cloudwork.order.domain.OrderInfo;
import com.snowvictor.cloudwork.order.dto.OrderCreateResult;
import com.snowvictor.cloudwork.order.mapper.OrderInfoMapper;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

/**
 * 独立 Bean: 仅承担订单的本地事务持久化。
 * 必须独立于 OrderService —— 如果写成 OrderService 的私有方法, createOrder()
 * 直接调用就是 self-invocation, Spring AOP 代理被绕开, @Transactional 不会生效。
 *
 * 事务边界: 仅覆盖 insert, 不包含上游的 Feign 调用 (避免长事务持锁)。
 */
@Service
@RequiredArgsConstructor
public class OrderPersistenceService {

    private final OrderInfoMapper orderMapper;

    @Transactional(rollbackFor = Exception.class)
    public OrderCreateResult persistAndReturn(OrderInfo info, StockDeductResponse resp) {
        orderMapper.insert(info);
        return new OrderCreateResult(
                info.getOrderId(),
                info.getStatus(),
                info.getUserId(),
                info.getUserName(),
                info.getProductId(),
                info.getProductName(),
                info.getQuantity(),
                info.getPrice(),
                info.getTotalAmount(),
                info.getFallbackMessage(),
                resp.getData()
        );
    }
}

package com.snowvictor.cloudwork.order.service;

import com.snowvictor.cloudwork.api.stock.dto.StockDeductRequest;
import com.snowvictor.cloudwork.api.stock.dto.StockDeductResponse;
import com.snowvictor.cloudwork.order.client.StockClient;
import com.snowvictor.cloudwork.order.domain.OrderInfo;
import com.snowvictor.cloudwork.order.dto.OrderCreateResult;
import com.snowvictor.cloudwork.order.dto.OrderRequest;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import java.math.BigDecimal;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;
import java.util.UUID;

@Service
@Slf4j
@RequiredArgsConstructor
public class OrderService {

    private final StockClient stockClient;
    private final OrderPersistenceService orderPersistenceService;   // 独立 Bean, 触发 Spring 代理使 @Transactional 生效

    /**
     * 创建订单。事务边界由独立 Bean OrderPersistenceService.persistAndReturn 承担,
     * Feign 调用在事务外执行 (避免长事务持锁)。
     */
    public OrderCreateResult createOrder(
            OrderRequest request, String userId, String userName, String userRole, String traceId
    ) {
        log.info("[{}] 开始创建订单: userId={}, productId={}, quantity={}",
                traceId, userId, request.getProductId(), request.getQuantity());

        StockDeductRequest stockRequest = new StockDeductRequest();
        stockRequest.setProductId(request.getProductId());
        stockRequest.setQuantity(request.getQuantity());

        // ─── Feign 调用在事务外 ─────────────────────────────────────────
        StockDeductResponse stockDeductResponse = stockClient.deductStock(
                userId, userName, userRole, traceId, stockRequest);

        if (stockDeductResponse == null) {
            log.error("[{}] 库存服务无响应", traceId);
            throw new IllegalStateException("库存服务无响应");
        }

        String orderId = generateOrderId();
        BigDecimal totalAmount = request.getPrice()
                .multiply(BigDecimal.valueOf(request.getQuantity()));

        if (stockDeductResponse.getCode() == 200) {
            Integer remaining = stockDeductResponse.getData() != null
                    ? stockDeductResponse.getData().getRemainingStock() : null;
            OrderInfo info = buildOrderInfo(orderId, "CREATED", userId, userName,
                    request, totalAmount, remaining, null, traceId);
            return orderPersistenceService.persistAndReturn(info, stockDeductResponse);
        }
        if (stockDeductResponse.getCode() == 503) {
            String fallbackMessage = "库存服务暂不可用，订单已降级创建，请后续人工核对库存";
            OrderInfo info = buildOrderInfo(orderId, "CREATED_FALLBACK", userId, userName,
                    request, totalAmount, null, fallbackMessage, traceId);
            return orderPersistenceService.persistAndReturn(info, stockDeductResponse);
        }
        throw new IllegalArgumentException(stockDeductResponse.getMessage());
    }

    private OrderInfo buildOrderInfo(String orderId, String status, String userId, String userName,
                                     OrderRequest req, BigDecimal totalAmount,
                                     Integer remainStock, String fallbackMessage, String traceId) {
        OrderInfo info = new OrderInfo();
        info.setOrderId(orderId);
        info.setStatus(status);
        info.setUserId(userId);
        info.setUserName(userName);
        info.setProductId(req.getProductId());
        info.setProductName(req.getProductName());
        info.setQuantity(req.getQuantity());
        info.setPrice(req.getPrice());
        info.setTotalAmount(totalAmount);
        info.setRemainStock(remainStock);
        info.setFallbackMessage(fallbackMessage);
        info.setTraceId(traceId);
        return info;
    }

    private String generateOrderId() {
        return "ORD" +
                LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss")) +
                UUID.randomUUID().toString().substring(0, 4).toUpperCase();
    }
}

## 第 N 章 · MySQL 与 MyBatis-Plus 接入

### 1. 目标

把 stock-service 与 order-service 从内存 `ConcurrentHashMap` 迁移到 MySQL 持久化，覆盖：

- MyBatis-Plus 实体与 Mapper
- Flyway 自动迁移
- 一实例多 schema + 双账号
- 乐观锁 + 悲观锁双策略，Nacos 热切换
- REQUIRES_NEW 独立事务做冲突审计

### 2. 数据模型（速览）

参见 `docs/superpowers/specs/2026-05-27-mysql-integration-design.md` §3 完整 DDL。

### 3. 策略模式：双锁可切换

```java
public interface StockDeductStrategy {
    LockMode mode();
    DeductOutcome deduct(String productId, int quantity);
}
```

- `OptimisticDeductStrategy`：依赖 `@Version`，`updateById` 自动追加 `WHERE version = ?`
- `PessimisticDeductStrategy`：`SELECT ... FOR UPDATE` 持行锁 + 手写 `UPDATE` 直接扣减
- `StockDeductStrategyResolver`：`@RefreshScope` + 构造时按 `mode().name()` 自建 Map（不依赖 bean name）

切换方式：

```yaml
# Nacos: stock-service-dev.yml
stock:
  deduct:
    lock-mode: PESSIMISTIC   # 默认 OPTIMISTIC, 改完保存即生效
```

### 4. 冲突审计：REQUIRES_NEW 独立事务

主事务方法 `StockService.deductStock` 抛 `OptimisticLockConflictException` 时，外层事务会回滚。为保留审计，**独立 Bean** `StockDeductAuditService.writeConflictLog(...)` 用 `@Transactional(REQUIRES_NEW)` 提交一条 `result_code = 409` 流水。

> ⚠️ 必须独立 Bean — 同类内私有方法因 Spring AOP self-invocation 不生效。

### 5. 双锁压测对比脚本

```sql
-- 跑完 IT 后, 在 stock_db 直接对比:
SELECT lock_mode,
       COUNT(*)                                AS attempts,
       SUM(CASE WHEN result_code = 200 THEN 1 ELSE 0 END) AS ok,
       SUM(CASE WHEN result_code = 409 THEN 1 ELSE 0 END) AS conflicts,
       AVG(cost_ms)                            AS avg_ms,
       MAX(cost_ms)                            AS max_ms
FROM stock_deduct_log
GROUP BY lock_mode;
```

### 6. 常见问题

- **冷启动报 `Table 'stock_db.stock_item' doesn't exist`**：检查 Flyway 是否启用（`spring.flyway.enabled=true`）以及账号权限。
- **乐观锁永远 409**：检查 `@Version` 字段是否加在 `version` 上，以及 `OptimisticLockerInnerInterceptor` 是否注册到 `MybatisPlusInterceptor`。
- **悲观锁出现死锁**：确认 `SELECT FOR UPDATE` 走的是 `uk_product_id` 唯一索引，不是表锁。
